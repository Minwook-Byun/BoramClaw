> **철학**: Bitter Lesson - "스캐폴딩을 쌓지 말고, 학습 가능한 구조를 만들어라"

## 측정 기준

### 좋은 프롬프트의 조건
1. **명확성**: 재시도 없이 원하는 결과
2. **간결성**: 불필요한 말 없이 핵심만
3. **재현성**: 같은 패턴을 다시 쓸 수 있음
4. **측정 가능**: 결과 품질을 객관적으로 판단 가능

## 프롬프트 품질 로그

### 2026-02-18

#### ✅ 고품질 프롬프트: "이번주 회고 1만자 + Karpathy 스킬"
```
이번주 리포트 출력 + 근데 네가 추가로 위 스킬도 써서 1만자 정도의 리포트 작성부탁
```

**왜 좋았나**:
- ✅ 구체적 분량 (1만자)
- ✅ 맥락 참조 (위 스킬 = Karpathy)
- ✅ 명확한 목표 (주간 회고)
- ✅ 재시도 0회

**결과**:
- 10,247자 (목표 달성)
- 10개 파트 구조화
- Karpathy 원칙 적용 확인
- 만족도: ⭐⭐⭐⭐⭐

**재사용 가능한 패턴**:
```
[기간] [문서 유형] + [특정 스킬/방법론] + [분량 제약]
```

---

#### ✅ 고품질 프롬프트: "링크드인 글자수 제한"
```
v3 한번 만들어보자. 링크드인 글자수 제한이 있잖아 ㅋㅋㅋ 코드 등은 빼고 간단 간단하게
```

**왜 좋았나**:
- ✅ 제약 조건 명시 (글자수 제한)
- ✅ 구체적 배제 사항 (코드 빼기)
- ✅ 톤 지정 (간단하게)
- ✅ 버전 관리 (v3)

**결과**:
- 2,965자 (3,000자 이하)
- 코드 제거됨
- 핵심 메시지 유지
- 만족도: ⭐⭐⭐⭐⭐

**재사용 가능한 패턴**:
```
[버전] + [플랫폼 제약] + [배제 요소] + [톤]
```

---

#### ⚠️ 개선 필요: "보안 꼼꼼하게"
```
보안상에 문제는 거의 없는 거지? 꼼꼼하게 한 번 살펴봐줘 중요한 문제니까
```

**왜 애매했나**:
- ⚠️ "꼼꼼하게"가 주관적
- ⚠️ "중요한 문제"의 범위 불명확
- ❌ 비교 대상 없음 (나중에 OpenClaw 비교 추가됨)

**실제 결과**:
- 9개 섹션 리뷰 (충분히 꼼꼼함)
- OpenClaw 비교 자동 추가됨 (맥락에서 추론)
- 만족도: ⭐⭐⭐⭐☆

**개선 방향**:
```
"OpenClaw와 비교해서 보안 리스크 분석해줘.
특히: API 키 관리, 네트워크 공격 표면, 데이터 프라이버시"
```
→ 구체적 항목 명시

## 발견된 패턴

### 패턴 1: "제약 기반 요청"
```
[작업] + [제약 조건] + [배제 요소]
```
예: "링크드인 포스트 3000자 이하, 코드 제외"

### 패턴 2: "맥락 참조"
```
[작업] + "위 [대상]" + [추가 지시]
```
예: "위 스킬도 써서 리포트 작성"
→ 대화 맥락 활용

### 패턴 3: "버전 관리"
```
"v[N] [요구사항]"
```
예: "v3 만들어보자"
→ 반복 개선 신호

### 패턴 4: "톤 지정"
```
[작업] + [톤 형용사]
```
예: "간단 간단하게", "꼼꼼하게", "진중하게"
→ 출력 스타일 제어
